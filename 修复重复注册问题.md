# 修复 SparseConv2d 重复注册问题

## 问题描述
运行时出现错误：
```
KeyError: 'SparseConv2d is already registered in conv layer'
```

## 根本原因
在 `mmdet3d/ops/spconv/conv.py` 文件中，所有 sparse conv 类都使用了 `@CONV_LAYERS.register_module()` 装饰器。这会导致：
1. 当模块被导入时，类会被自动注册到 `CONV_LAYERS` 注册表
2. 如果同一模块被多次导入或在不同地方被注册，就会触发 "already registered" 错误
3. 这些类实际上是通过直接导入使用的（如 `from mmdet3d.ops.spconv import SparseConv2d`），并不需要注册到 `CONV_LAYERS`

## 解决方案
**移除 `mmdet3d/ops/spconv/conv.py` 中所有类的 `@CONV_LAYERS.register_module()` 装饰器**

## 需要修改的类（共 10 个）
1. `SparseConv2d` (约第 227 行)
2. `SparseConv3d` (约第 255 行)
3. `SparseConv4d` (约第 283 行)
4. `SparseConvTranspose2d` (约第 311 行)
5. `SparseConvTranspose3d` (约第 339 行)
6. `SparseInverseConv2d` (约第 369 行)
7. `SparseInverseConv3d` (约第 383 行)
8. `SubMConv2d` (约第 397 行)
9. `SubMConv3d` (约第 426 行)
10. `SubMConv4d` (约第 455 行)

## 修改方法

### 方法 1：使用 sed 命令（推荐）
在服务器上执行以下命令：

```bash
cd /root/workbench/test_bevfusion

# 备份原文件
cp mmdet3d/ops/spconv/conv.py mmdet3d/ops/spconv/conv.py.bak

# 移除所有 @CONV_LAYERS.register_module() 装饰器
sed -i '/@CONV_LAYERS\.register_module()/d' mmdet3d/ops/spconv/conv.py

# 验证修改
grep -n "@CONV_LAYERS.register_module()" mmdet3d/ops/spconv/conv.py
# 应该没有输出
```

### 方法 2：手动编辑
使用 vim 或其他编辑器手动删除所有 `@CONV_LAYERS.register_module()` 行。

## 为什么之前可能失败

1. **代码未同步**：本地修改没有应用到服务器
2. **缓存问题**：Python 的 `__pycache__` 目录可能包含旧的编译文件
3. **多环境冲突**：可能存在多个 Python 环境或包路径

## 如果问题仍然存在

执行以下清理操作：

```bash
cd /root/workbench/test_bevfusion

# 清理所有 Python 缓存
find . -type d -name "__pycache__" -exec rm -rf {} + 2>/dev/null
find . -type f -name "*.pyc" -delete 2>/dev/null

# 确认修改已应用
grep -c "@CONV_LAYERS.register_module()" mmdet3d/ops/spconv/conv.py
# 应该输出 0

# 如果还是不行，尝试重新安装 mmdet3d
pip uninstall mmdet3d -y
pip install -e .
```

## 技术说明

- `CONV_LAYERS` 是 `mmcv.cnn` 提供的注册器，用于注册卷积层
- 使用 `@register_module()` 装饰器后，类会在导入时自动注册
- 如果类只通过直接导入使用（`from x import y`），不需要注册
- 重复注册会触发 `KeyError: 'xxx is already registered'` 错误
