# 修复 SparseConv2d 重复注册问题 (最终解决方案)

## 问题描述
运行时出现错误：
```
KeyError: 'SparseConv2d is already registered in conv layer'
```

## 根本原因
经过深入分析，问题的根本原因是：

1. **双重依赖**：
   - `sparse_block.py` 使用 `build_conv_layer` 函数从 `CONV_LAYERS` 注册表中查找卷积层
   - 同时 `conv.py` 中的类使用 `@CONV_LAYERS.register_module()` 装饰器进行注册

2. **重复导入**：
   - 当模块被多次导入时，装饰器会重复执行，导致 "already registered" 错误

## 最终解决方案
**使用条件注册机制**：保留装饰器但确保只注册一次

## 技术原理

1. **条件注册**：使用 `if not hasattr(CONV_LAYERS, '_spconv_registered'):` 确保注册代码只执行一次

2. **注册表标记**：注册完成后设置 `CONV_LAYERS._spconv_registered = True`，防止重复执行

3. **保持兼容性**：保留了 `@CONV_LAYERS.register_module()` 装饰器，确保 `build_conv_layer` 函数能正常工作

## 为什么这个方案有效

- **解决了重复注册**：条件检查确保装饰器只执行一次
- **保持了功能完整**：保留了装饰器，确保 `sparse_block.py` 中的 `build_conv_layer` 能找到所需的类
- **通用性强**：适用于任何可能导致重复导入的场景
