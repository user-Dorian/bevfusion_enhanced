# 最终修复指南 - spconv 重复注册问题

## ⚠️ 问题症状

```
KeyError: 'SparseConv2d is already registered in conv layer'
```

## 🔍 根本原因

**服务器上的代码没有同步到最新版本！**

错误信息显示服务器仍在使用旧版代码：
- 旧版：`from mmdet3d.ops import SparseBasicBlock` (会触发重复注册)

## 💡 技术说明

**为什么需要这样修改？**

1. **问题根源**：`mmdet3d/ops/__init__.py` 导入 `sparse_block` 时，会触发 `spconv/conv.py` 中的 `@CONV_LAYERS.register_module()` 装饰器

2. **重复注册**：如果之前已有其他地方（如 `iou3d` 模块）导入过 `spconv`，`SparseConv2d` 就会被注册两次，导致 KeyError

3. **解决方案**：
   - 注释掉 `__init__.py` 中的导入，避免初始化时触发注册
   - 在需要使用时直接导入（`from mmdet3d.ops.sparse_block import ...`）
   - 这样每个模块只会在第一次导入时注册一次

## ⚠️ 常见问题

### Q1: 为什么 git pull 后还是报错？

**可能原因**：
- 服务器上的代码有本地修改，git 无法自动合并
- 需要使用 `git reset --hard origin/main` 强制覆盖

### Q2: 修改后仍然报错怎么办？

**解决方案**：
1. 确认所有 3 个文件都已正确修改
2. 清除 Python 缓存（`__pycache__` 和 `.pyc` 文件）
3. 重启 Python 解释器或终端

### Q3: NumbaDeprecationWarning 警告需要处理吗？

**不需要**。这只是警告，不影响运行。可以忽略。
